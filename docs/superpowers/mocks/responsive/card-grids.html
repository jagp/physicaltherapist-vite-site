Card grids: container queries + subgrid

The same identical markup is dropped into three different-width slots. Cards count their container, not the viewport — 1-up in a sidebar-narrow slot, 2-up at tablet, 3-up in the full content column. Row internals align via subgrid (watch the “Learn more” links line up even with different body lengths).

Why this matters: when the same ServiceCard later lands in a narrower “related services” rail or a wider feature band, it adapts with zero new CSS — the component owns its responsiveness. Both container queries and subgrid are Baseline Widely available (mid-2026); the ~7% legacy tail simply sees non-aligned rows, never broken layout.
